hash路由和history路由 ​

hash模式 ​

当 URL 的 hash 部分发生变化时，浏览器会触发 hashchange 事件，路由库会监听这个事件并根据新的 hash 值来渲染相应的组件。

history模式 ​

使用 History API 中的 pushState 和 replaceState 方法来改变 URL，同时监听 popstate(前进后退) 事件以便在 URL 发生变化时进行相应的路由切换
